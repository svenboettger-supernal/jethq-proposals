Workspace · 03

SOP Library

Every standard operating procedure the company keeps on Microsoft 365, pulled into one searchable library inside Flight Deck. A summer intern is lined up to help with the migration.

01 / Overview

The brief

SOPs today are scattered across Microsoft 365. The team wants them in one place, searchable, inside the platform they already open. This is a low-effort, high-value early win.

A summer intern is available to pair with Supernal on the migration, so the lift on JetHQ is light.

02 / Objectives

What success looks like

  • One searchable SOP library that lives inside Flight Deck.
  • SOP content migrated out of Microsoft 365 and into the library.
  • Clear categories so every procedure has a place.
  • An owner and a last-updated date on every SOP.
  • Find the right procedure fast, without hunting across folders.
  • A simple flow to add and update SOPs as the company changes.

03 / Scope

Scope of work

Stage 1 · Phase 2

Structure and core migration

Set the library structure and get the core SOPs out of Microsoft 365 and in.

  • Stand up the library structure and categories inside Flight Deck.
  • Migrate the core SOPs from Microsoft 365.
  • Confirm the category structure with JetHQ before content lands.

Stage 2 · Phase 2

Search and ownership

Make the library fast to search and clear on who owns what.

  • Search and filtering across the full library.
  • An owner on every SOP.
  • A last-updated date on every SOP.

Stage 3 · Phase 2

Full library and upkeep

Finish the migration and give the team a simple way to keep it current.

  • The full library migrated from Microsoft 365.
  • A simple flow to add new SOPs.
  • A simple flow to update existing SOPs.
