Package 05

Sales Ops

Real-time dashboards connected to Salesforce. Inbound leads, pipeline, forecast, comp, leaderboards, and deal-health flags in one place, with listings sourced only from the Hub.

01 / Overview

The brief

Salesforce is used the traditional way, deals, contacts, and companies. The CRM manager can grant API access, so Flight Deck pulls the data in and visualizes it, cleaner and more relevant than the raw Salesforce views.

The first step is an audit of the Salesforce instance itself. Before anything is visualized, Supernal maps the standard and custom objects and fields in use and confirms the API endpoints expose everything Flight Deck needs. Salesforce stays the system of record in this phase, so any field it shares with the Hub is matched deliberately and nothing is lost in translation. Whether Salesforce is retired later is a separate, future decision.

02 / Objectives

What success looks like

  • Real-time results replace the static Yearly Results, so the sales picture is current the moment it is opened.
  • Pipeline is pulled from Salesforce by API and visualized cleaner than the raw CRM views.
  • Deal-health flags surface deals that are stalling or at risk before they slip.
  • Comp and leaderboards keep performance visible against goal.
  • Forecast rolls up from the live pipeline instead of a manual spreadsheet.

03 / Scope

Scope of work

Stage 1 · Q4 2026

Salesforce connection and real-time results

Audit what is inside Salesforce, connect the CRM, and replace the static Yearly Results with a live dashboard.

  • Audit the Salesforce instance first: standard and custom objects, custom fields, and confirm the API endpoints expose everything Flight Deck needs.
  • Match any field shared with the Hub so the two stay consistent.
  • Connect Salesforce by API with access granted by the CRM manager.
  • Pull deals, contacts, and companies into Flight Deck.
  • Stand up the real-time results dashboard that replaces Yearly Results.

Stage 2 · Q4 2026

Inbound leads and pipeline

Surface inbound leads and pipeline from what Salesforce already holds, and make deal health readable at a glance.

  • Surface inbound leads from Salesforce, using the source and activity data the instance already captures.
  • Build the pipeline view from the Salesforce data.
  • Flag deals that are stalling or at risk, using the criteria JetHQ defines.

Stage 3 · Q4 2026

Comp and forecast

Round out the performance views with comp, leaderboards, and forecast.

  • Comp and leaderboards against goal.
  • Forecast rolled up from the live pipeline.
